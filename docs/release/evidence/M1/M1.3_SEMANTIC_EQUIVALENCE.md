## Semi-Continuous Partial-Apply Counterexample

**Current behavior (P1 defect):** When HiGHS applies an ordinary bound change but rejects a subsequent semi-continuous domain operation as unsupported, the `sync_model` drain loses both the applied and rejected operations. No replayable batch remains.

**Fix (M1.3):** With the P2 revision protocol:
1. Model mutations produce `DeltaBatch` values stored in the `Journal`
2. Each `AdapterCursor` tracks its applied revision independently
3. On `RequiresRebuild` (from unsupported operation), the cursor marks `needs_rebuild`
4. The coordinator provides the snapshot at the cursor's revision for rebuild
5. After rebuild, the cursor advances past the problematic revision
6. No delta is ever destroyed — the journal retains everything until compaction

**Status:** Protocol defined. Implementation requires wiring the P2 types into the actual HiGHS adapter (M1.2+). The reference backend proves the protocol works — the semi-continuous scenario is modeled in the differential harness.
