## Build Modes

1. **Bundled static (default):** `highs-sys` 1.15.0 builds HiGHS from bundled source via cmake.
   No pre-installed HiGHS required. Preferred for portability.
2. **System discovery:** Link against system-installed HiGHS via `HIGHS_ROOT` or `HIGHS_LIB_DIR`.
   ROML's `build.rs` handles library discovery. Optional feature.

## docs.rs Compatibility

`roml-highs` requires native HiGHS libraries at build time. docs.rs may not be able
to build the crate. Options:
1. Use `[package.metadata.docs.rs]` to set `default-features = false` or provide
   a `docs-rs` feature that stubs native functions.
2. Document that roml-highs docs are built locally.

Decision deferred to M1.8 (release qualification).

## Sanitizer/Fuzz/Miri Lanes

Deferred. The unsafe boundary in roml-highs is now isolated to `highs-sys` which
owns the FFI contract. ROML no longer contains handwritten ABI declarations.

- **Miri:** Not applicable (FFI boundary)
- **Sanitizers:** Address/leak/undefined behavior sanitizers should be run against
  the adapter with HiGHS linked. CI infrastructure needed.
- **Fuzz:** Mutation sequence fuzzing via the differential harness (M1.3).
  Generated sequences can be replayed against both ReferenceBackend and HiGHS.
