  <action>
    Add a `pub cursor: AdapterCursor` field to the `ReferenceBackend` struct so the BackendSession impl owns its own cursor state (per D1). The existing `apply_batch(&mut self, batch, cursor)` and `rebuild(&mut self, snapshot, cursor)` methods remain unchanged for backward compat with `tests/backend_contract.rs`.

    Add `impl BackendSession for ReferenceBackend`:
    - `synchronize(&mut self, sync: Synchronization) -> Result<SyncReceipt, BackendError>`:
      - `Synchronization::DeltaBatch(batch)`: call `self.apply_batch(&batch, &mut self.cursor)`. Convert the returned ApplyOutcome:
        - `ApplyOutcome::Applied { .. }` -> `Ok(SyncReceipt { cursor: self.cursor.clone(), health: AdapterHealth::Ready })`
        - `ApplyOutcome::RecoverableFailure { reason }` -> `Err(BackendError::new(reason, ErrorCategory::InvalidInput, HealthEffect::Recoverable))`
        - `ApplyOutcome::RequiresRebuild { reason, .. }` -> `Err(BackendError::new(reason, ErrorCategory::Unsupported, HealthEffect::RequiresRebuild))`
        - `ApplyOutcome::DirtyFailure { reason }` -> `Err(BackendError::new(reason, ErrorCategory::Internal, HealthEffect::RequiresRebuild))`
      - `Synchronization::Rebuild(snapshot)`: call `self.rebuild(&snapshot, &mut self.cursor)`. Return `Ok(SyncReceipt { cursor: self.cursor.clone(), health: AdapterHealth::Ready })`.
      - Cursor apply_batch may return `Err(String)` from cursor.advance failure -> convert to `BackendError::new(.., ErrorCategory::Internal, HealthEffect::Terminal)`.
    - `solve(&mut self, _request: &SolveRequest) -> Result<SolveResult, BackendError>`: return `Err(BackendError::new("ReferenceBackend does not support solve operations", ErrorCategory::Unsupported, HealthEffect::None))`.
    - `close(self) -> Result<(), BackendError>`: return `Ok(())`.

    Add `impl SessionHealth for ReferenceBackend` returning cursor.health / cursor.applied_revision.

    Add `impl BackendMetadata for ReferenceBackend`:
    - `name()` returns "ReferenceBackend"
    - `capabilities()` returns `BackendCapabilities` with all structural flags true (add_variable, add_constraint, set_coefficient, set_bounds, set_objective, delete, semicontinuous, semiinteger, parameter_update) and all solve flags false (lp, mip, callbacks, solution, duals, reduced_costs).

    Re-export imports needed: the file already has `use crate::sync::AdapterCursor` but needs `use crate::solver::backend::{BackendCapabilities, BackendError, ErrorCategory, HealthEffect}`, `use crate::solver::request::SolveRequest`, and `use crate::solver::session::{BackendMetadata, BackendSession, SessionHealth, SyncReceipt, Synchronization}`.

    Do NOT modify any existing method signatures or their callers. Do NOT import unused names.

  <action>
    Add `BackendFixture` trait to `src/solver/session.rs` (per D2):
    ```rust
    pub trait BackendFixture {
        type Session: BackendSession;
        fn new_session(&self) -> Result<Self::Session, BackendError>;
        fn backend_name(&self) -> &str;
    }
    ```
    Do not gate behind any feature. The trait is always public.

    Add `pub struct RefBackendFixture;` and `impl BackendFixture for RefBackendFixture` at the end of `src/solver/reference.rs`. It creates a `ReferenceBackend::new()` in new_session and returns "ReferenceBackend" from backend_name.

    Create `src/solver/conformance.rs` as a new module with `pub fn run_sync_suite<F: BackendFixture>(fixture: &F)`. This function runs a parameterized synchronization conformance suite that every backend must pass. Include these scenarios:

    1. **empty_rebuild**: Rebuild from `ModelSnapshot::empty(r0)`. Assert SyncReceipt health == Ready, cursor revision == r0.
    2. **full_rebuild**: Rebuild from a snapshot with one variable (NON_NEGATIVE, Continuous, active) and one constraint (le 100.0, active). Assert health Ready, cursor at snapshot revision.
    3. **single_delta_apply**: Start from empty rebuild, apply a DeltaBatch with one AddVariable operation. Assert health Ready, cursor advances.
    4. **multi_batch_sequence**: Three sequential delta batches adding variable, constraint, cell. Assert each advances cursor, final state Ready.
    5. **revision_mismatch_error**: Rebuild to r1, apply a batch whose `from` revision does not equal the cursor's applied_revision. Assert BackendError (Recoverable health effect).
    6. **rebuild_resets_state**: Apply some deltas to populate state, then rebuild from empty snapshot. Assert cursor at the rebuild revision.
    7. **close_after_rebuild**: Call close() and assert Ok(()).

    Each scenario uses `fixture.new_session()` to create a fresh session, calls `synchronize` / `solve` / `close` via the `BackendSession` trait, and uses `std::assert_eq!`/`std::assert!`/`std::panic!` directly (no test framework macros needed).

    Add `pub mod conformance;` to `src/solver/mod.rs` so the module is public (needed by integration tests in both crates). Place the declaration before the other module declarations or after them — any order is fine as long as `pub mod conformance;` is present.

    Do NOT wrap conformance.rs in `#[cfg(test)]` — integration tests in other crates need access. The function bodies use standard assertions which compile in all modes.
