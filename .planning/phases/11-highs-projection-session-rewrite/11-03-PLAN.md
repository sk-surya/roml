      ```

      ```

      - If run_status == kHighsStatusWarning, log a warning but proceed to model status mapping.
      - If run_status == kHighsStatusOk, proceed normally.

    2. **Model status mapping** (exhaustive per AD-6 table):

      - MODEL_STATUS_OPTIMAL (7) -> TerminationStatus::Optimal
      - MODEL_STATUS_INFEASIBLE (8) -> TerminationStatus::Infeasible
      - MODEL_STATUS_UNBOUNDED_OR_INFEASIBLE (9) -> TerminationStatus::InfeasibleOrUnbounded

      - MODEL_STATUS_UNBOUNDED (10) -> TerminationStatus::Unbounded
      - MODEL_STATUS_OBJECTIVE_BOUND (11) -> check feasible incumbent; Feasible if found else Error
      - MODEL_STATUS_OBJECTIVE_TARGET (12) -> check feasible incumbent; Feasible if found else Error
      - MODEL_STATUS_TIME_LIMIT (13) -> TerminationStatus::TimeLimit
      - MODEL_STATUS_ITERATION_LIMIT (14) -> TerminationStatus::IterationLimit
      - MODEL_STATUS_MODEL_EMPTY (6) -> TerminationStatus::Optimal
      - MODEL_STATUS_LOAD_ERROR (1) -> TerminationStatus::Error
      - MODEL_STATUS_MODEL_ERROR (2) -> TerminationStatus::Error

      - Allocate vec for primal values.
      - Call `Highs_getSolution(raw, primal.as_mut_ptr(), dual.as_mut_ptr(), ptr::null_mut())`.
      - Return true if the call succeeds and primal contains valid values (non-NaN).

    4. **Interruption check** (AD-8):
      - If the user set a callback handler and it returned Interrupt, and the run returned with e.g., TIME_LIMIT or ITERATION_LIMIT, map to TerminationStatus::Interrupted.
      - For now, this is checked via the callback handler's interrupt flag stored in CallbackState (accessible from session.rs level). At the solution.rs level, we don't track the interrupt flag — it's handled in session.rs logic.

    - Call `Highs_getSolution(raw, primal_ptr, dual_ptr, reduced_cost_ptr)`.

    - Map dual values: iterate row_map to map HiGHS row index -> ConId -> f64.
    - Build SolveSolution with values, duals, reduced_costs, objective_value.
    - For MIP solutions (status is Feasible) — duals and reduced costs may not be available. Return only what Highs_getSolution provides.

    For ffi safety (CString):
    - When calling Highs_getInfoValue for iteration count or node count, construct CString properly and handle CString::new() failures.
