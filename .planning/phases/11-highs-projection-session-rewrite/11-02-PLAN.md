  <action>
    Create projection.rs implementing deterministic full rebuild of HiGHS model from a ModelSnapshot. The module has no public struct — it exposes free functions that operate on the HighsSession's raw handle and caches.

    Required imports:
    - `crate::bindings::*`, `crate::error::check_highs_status`, `crate::index_map::IndexMap`
    - `roml::snapshot::{ModelSnapshot, VariableEntry, ConstraintEntry, CellEntry, ObjectiveEntry, ParameterEntry}`
    - `roml::model::coefficient::{CellKey, CoefficientTarget}`
    - `roml::model::{Bounds, ConstraintBounds, VarType, Sense}`
    - `roml::id::{VarId, ConId, ObjId, ParamId}`
    - `roml::solver::backend::BackendError`
    - `std::collections::HashMap`, `std::ffi::CString`

    Function `pub(crate) fn check_semicontinuous(snapshot: &ModelSnapshot) -> Result<(), BackendError>`:
    - Iterate `snapshot.variables`. If any entry has `semicontinuous_lower: Some(_)`, return `Err(BackendError::unsupported("semi-continuous variables"))`.
    - Per M1R-H7, this must happen BEFORE any state modification. No HiGHS calls are made before this check.

    Function `pub(crate) fn rebuild_from_snapshot(
        raw: *mut c_void,
        snapshot: &ModelSnapshot,
        col_map: &mut IndexMap<VarId>,
        row_map: &mut IndexMap<ConId>,
        inf: f64,
        var_bounds: &mut HashMap<VarId, (f64, f64)>,
        con_bounds: &mut HashMap<ConId, (f64, f64)>,
        obj_costs: &mut HashMap<ObjId, HashMap<VarId, f64>>,
        obj_senses: &mut HashMap<ObjId, Sense>,
        active_obj: &mut Option<ObjId>,
    ) -> Result<(), BackendError>`:

    Sequence per Research Architecture Patterns:
    1. Call `check_semicontinuous(snapshot)?` — reject before any modification.
    2. Clear HiGHS model: `unsafe { Highs_clear(raw) }`. Check return code.
    3. Clear all maps (col_map, row_map, var_bounds, con_bounds, obj_costs, obj_senses, *active_obj = None).
    4. Add variables: For each `VariableEntry` in `snapshot.variables`:
       - Normalize bounds: `fn highs_bound(b: f64, inf: f64) -> f64` mapping f64::INFINITY/NEG_INFINITY to inf/-inf.
       - Call `unsafe { Highs_addVar(raw, lb, ub) }`. Check return code. Assign returned index to `col_map.insert(var.id, idx)`.
       - If var_type is Integer or Binary: `unsafe { Highs_changeColIntegrality(raw, idx, type_code) }`. Check return code.
       - If inactive: `unsafe { Highs_changeColBounds(raw, idx, 0.0, 0.0) }`. Check return code.
       - Cache bounds: `var_bounds.insert(var.id, (normalized_lb, normalized_ub))`.
    5. Add constraints: For each `ConstraintEntry` in `snapshot.constraints`:
       - Normalize constraint bounds.
       - Call `unsafe { Highs_addRow(raw, lb, ub, 0, ptr::null(), ptr::null()) }` (add empty row first). Check return code. Assign `row_map.insert(con.id, idx)`.
       - Cache bounds: `con_bounds.insert(con.id, (normalized_lb, normalized_ub))`.
    6. Add cells: For each `CellEntry` where target is a constraint:
       - Look up row index via row_map, col index via col_map.
       - Call `unsafe { Highs_changeCoeff(raw, row, col, value) }`. Check return code.
    7. Add objectives and cells: For each `ObjectiveEntry`:
       - `obj_senses.insert(obj.id, obj.sense)`; `obj_costs.insert(obj.id, HashMap::new())`.
       - For each objective cell (CellKey with CoefficientTarget::Objective): insert into obj_costs cache and call `Highs_changeColCost(raw, col, cost)`. Check return code.
       - Set objective sense: `unsafe { Highs_changeObjectiveSense(raw, sense_code) }`. Check return code.
       - If active: `*active_obj = Some(obj.id)`.
    8. Set active objective: If `active_obj` is Some, set sense and costs.
    9. For inactive variables: bounds set to [0, 0] (already handled in step 4). For inactive constraints: bounds set to [-inf, inf] (unconstrained).
       - Handle via separate loop over inactive entities.

    Normalization helper (extracted from old adapter pattern):
    ```rust
    pub(crate) fn normalize_bound(b: f64, inf: f64) -> f64 {
        if b == f64::NEG_INFINITY { -inf }
        else if b == f64::INFINITY { inf }
        else { b }
    }
    ```

    Infinity handling per Pitfall 6: HiGHS uses 1e30 as internal infinity. f64::INFINITY causes HiGHS to reject calls. Always normalize bounds.

    Every Highs_addVar, Highs_addRow, Highs_changeCoeff, Highs_changeColIntegrality, Highs_changeColBounds, Highs_changeObjectiveSense call is wrapped with `check_highs_status(ret, raw, "op_name")?`.

  <name>Task 2: Delta Application — apply_delta_batch() and ModelOp handling</name>
  <files>

  <action>
    Implement delta application in projection.rs. All 16 ModelOp variants must be handled (or explicitly rejected with clear errors).

    Function `pub(crate) fn apply_delta_batch(
        raw: *mut c_void,
        batch: &DeltaBatch,
        col_map: &mut IndexMap<VarId>,
        row_map: &mut IndexMap<ConId>,
        inf: f64,
        var_bounds: &mut HashMap<VarId, (f64, f64)>,
        con_bounds: &mut HashMap<ConId, (f64, f64)>,
        obj_costs: &mut HashMap<ObjId, HashMap<VarId, f64>>,
        obj_senses: &mut HashMap<ObjId, Sense>,
        active_obj: &mut Option<ObjId>,
    ) -> Result<(), BackendError>`:

    Prevalidation phase: Iterate all operations first. If any op has semi-continuous data (e.g., AddVariable with a hypothetical semicontinuous field — check VariableEntry patterns), reject BEFORE any state modification with BackendError::unsupported per M1R-H7. Currently ModelOp has no semi-continuous fields, so this is mainly for future-proofing.

    Apply phase (atomic — acknowledge only on complete success):

    For each ModelOp variant:
    - `AddVariable { var, bounds, var_type }`:
      Normalize bounds, `Highs_addVar(raw, lb, ub)`, check code, col_map.insert, set integrality if needed, cache in var_bounds.
    - `RemoveVariable { var }`:
      Get idx from col_map, `Highs_deleteCols(raw, 1, &[idx])`, check code, col_map.remove(var), var_bounds.remove(var), call `col_map.reindex_after_delete(idx)`, remove from obj_costs cache.
    - `SetVariableBounds { var, bounds }`:
      Get idx, normalize bounds, `Highs_changeColBounds(raw, idx, lb, ub)`, check code, update var_bounds cache.
    - `SetVariableActive { var, active }`:
      Get idx, if !active: fix bounds to [0,0], else: restore from var_bounds cache via `Highs_changeColBounds(raw, idx, lb, ub)`.
    - `SetVariableType { var, var_type }`:
      Get idx, `Highs_changeColIntegrality(raw, idx, type_code)`, check code.
    - `AddConstraint { con, bounds }`:
      Normalize, `Highs_addRow(raw, lb, ub, 0, null, null)`, check code, row_map.insert, cache con_bounds.
    - `RemoveConstraint { con }`:
      Get idx, `Highs_deleteRows(raw, 1, &[idx])`, check code, row_map.remove(con), row_map.reindex_after_delete(idx), con_bounds.remove(con).
    - `SetConstraintBounds { con, bounds }`:
      Get idx, normalize, `Highs_changeRowBounds(raw, idx, lb, ub)`, check code, update cache.
    - `SetConstraintActive { con, active }`:
      Get idx, if !active: bounds to [-inf, inf] (unconstrained), else: restore from cache.
    - `SetCell { cell_key, evaluated_value, .. }`:
      Get row/col indices, `Highs_changeCoeff(raw, row, col, value)`, check code.
    - `RemoveCell { cell_key }`:
      Get row/col indices, `Highs_changeCoeff(raw, row, col, 0.0)`, check code. (HiGHS doesn't have a direct "remove coefficient" — setting to zero achieves the same effect.)
    - `AddObjective { obj, sense }`:
      Store in obj_senses, initialize empty costs map in obj_costs. No HiGHS API call — objectives are virtual in ROML.
    - `RemoveObjective { obj }`:
      Remove from obj_senses and obj_costs. If this was the active objective (active_obj == Some(obj)), set active_obj = None and zero all costs via `Highs_changeColsCostByRange(raw, 0, num_cols-1, zeros)`.
    - `SetActiveObjective { obj }`:
      Per Pitfall 5: Zero all costs first via `Highs_changeColsCostByRange(raw, 0, num_cols-1, &[0.0; num_cols])`, then set new costs from obj_costs cache. Update active_obj. Set sense via Highs_changeObjectiveSense.
    - `SetObjectiveCell { cell_key, evaluated_value, constant }`:
      Get col index, update obj_costs cache, call `Highs_changeColCost(raw, col, cost)`, check code. Handle constant separately — store in a separate map or note (used during extraction in Plan 03).
    - `SetParameter { param, value }`:
      Look up parameter mapping (if any HiGHS equivalent). If no HiGHS equivalent, skip (model-internal parameter). Use CString for names.

    Re-indexing per Pitfall 4: After every delete (RemoveVariable, RemoveConstraint), call `reindex_after_delete` on the corresponding map to keep dense indices in sync.

    Objective switching per Pitfall 5: When switching active objective, zero ALL column costs via `Highs_changeColsCostByRange` before loading the new objective's costs. This prevents stale costs from the previous objective blending with the new one.

    Return `Ok(())` only if every operation succeeded. If any operation fails, return the error from check_highs_status — do NOT partially acknowledge.

    Handle the `VariableEntry.semicontinuous_lower` field: If a ModelOp adds a variable with semi-continuous data (checking all 16 ModelOp variants for semicontinuous fields — currently none carry it), reject beforehand. The rejection preserves replayability per M1R-H7.

    pub(crate) struct CallbackState {
        pub handler: Box<dyn CallbackHandler>,
        pub col_map: *const IndexMap<VarId>,

        pub highs_ptr: *mut c_void,

    The callback trampoline (extracted from old adapter pattern):
    ```rust
    // SAFETY: Called from HiGHS C callback context. We only access data_out
    // through verified pointers. user_data is a Box<CallbackState> we created.
    pub(crate) unsafe extern "C" fn callback_trampoline(
        event_type: c_int,
        _message: *const c_char,
        data_out: *const HighsCallbackDataOut,
        _data_in: *mut HighsCallbackDataIn,
        user_data: *mut c_void,
    ) -> c_int {
        let state = &mut *(user_data as *mut CallbackState);
        match event_type {
            CALLBACK_MIP_LOGGING => { /* informational — log message if desired */ 0 },
            CALLBACK_MIP_INTERRUPT => {
                // Check user interruption via CallbackHandler
                // Return non-zero to trigger user_interrupt in HiGHS
                0
            },
            CALLBACK_MIP_SOLUTION => {
                // Informational only — map to CallbackEvent::CandidateSolution
                // Extract primal solution from data_out.mip_solution if available
                0
            },
            CALLBACK_MIP_IMPROVING_SOLUTION => {
                // Informational only — improving incumbent found
                // Map to CallbackEvent::ImprovingIncumbent
                0
            },
            CALLBACK_MIP_DEFINE_LAZY_CONSTRAINTS => {
                // Officially supported — invoke handler.on_event()
                // Use CallbackHandler::on_event with CallbackEvent::LazyConstraints
                // If handler returns InjectLazyConstraints, add rows via Highs_addRow
                0
            },
            CALLBACK_MIP_GET_CUT_POOL => {
                // Read-only diagnostic — NOT implementing cut parsing
                0
            },
            _ => 0, // Unknown callback type — safely ignore per HiGHS contract
        }
    }
    ```

    Per AD-4 callback disposition:
    - kCallbackMipLogging (1): Supported — informational only, log the message string.
    - kCallbackMipInterrupt (2): Supported — check handler for user_interrupt flag, set `data_in.user_interrupt`.
    - kCallbackMipSolution (3): Supported — informational only, report candidate solution.
    - kCallbackMipImprovingSolution (4): Supported — informational only, report improving incumbent.
    - kCallbackMipGetCutPool (7): Not implemented — read-only diagnostic, safely no-op.
    - kCallbackMipDefineLazyConstraints (8): Officially supported, implemented — invoke CallbackHandler, inject constraints via Highs_addRow.
    - User cuts and incumbent injection: REJECTED per AD-4 (not in official callback type list).

    Function `pub(crate) fn register_callback(
        raw: *mut c_void,
        handler: Box<dyn CallbackHandler>,
        col_map: *const IndexMap<VarId>,
        row_map: *const IndexMap<ConId>,
        num_cols: i32,
    ) -> Result<*mut CallbackState, BackendError>`:
    - Box::new(CallbackState { ... }) into raw pointer.
    - Call `Highs_setCallback(raw, Some(callback_trampoline as extern "C" fn(...)), state_ptr)`.
    - Check return code. Return the state pointer.

    Function `pub(crate) fn clear_callback(raw: *mut c_void, state: *mut CallbackState)`:
    - SAFETY: Reconstruct Box from state pointer to drop it.
    - Call `Highs_setCallback(raw, None, ptr::null_mut())`.
    - The state pointer must be valid and non-null.

    Function `pub(crate) fn collect_lazy_constraints(raw: *mut c_void, handler: &mut CallbackHandler)`:
    - If handler returns CallbackAction::InjectLazyConstraints, parse constraint data and call Highs_addRow.
    - Check each Highs_addRow return code.

    Thread safety per M1R-H4: CallbackState is created before solve and destroyed after. The SAFETY comment on callback_trampoline must document that access to CallbackState is single-threaded during solve. After solve, the state is cleaned up before any Send.

    Catch Rust panics: Use `std::panic::catch_unwind` around handler invocations in the trampoline to prevent panics crossing the C boundary. Log any panic and return 0 gracefully.

    Register the callback types by using the kCallbackMip* constants from bindings.rs.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HighsSession internal -> HiGHS C API | Projection calls: every addVar/addRow/changeCoeff is unfenced unsafe |
| HiGHS C callback -> Rust handler | Callback bridge: C calls into Rust with user_data pointer |
| native -> rust panic boundary | Catch_unwind in callback: Rust panic must not cross C stack frame |
